Based on the ERD, the **Prediction / Machine Learning layer** consists of the following tables:

# 1. pred_model_registry

Stores metadata about trained ML models.

## Columns

| Column           | Type         |
| ---------------- | ------------ |
| model_key        | BIGINT (PK)  |
| model_name       | VARCHAR(255) |
| model_version    | VARCHAR(50)  |
| model_type       | VARCHAR(100) |
| algorithm        | VARCHAR(100) |
| training_date    | TIMESTAMP    |
| deployment_date  | TIMESTAMP    |
| accuracy_score   | DECIMAL      |
| precision_score  | DECIMAL      |
| recall_score     | DECIMAL      |
| f1_score         | DECIMAL      |
| status           | VARCHAR(50)  |
| model_parameters | JSONB        |
| feature_list     | JSONB        |
| created_at       | TIMESTAMP    |
| updated_at       | TIMESTAMP    |

## Purpose

Tracks:

* Model versions
* Performance metrics
* Deployment status
* Training configurations

---

# 2. pred_time_series_forecast

Stores future forecasts generated by time-series models.

## Columns

| Column                  | Type          |
| ----------------------- | ------------- |
| forecast_key            | BIGINT (PK)   |
| model_key               | BIGINT (FK)   |
| account_key             | INT (FK)      |
| category_key            | INT (FK)      |
| user_key                | INT (FK)      |
| forecast_date_key       | INT (FK)      |
| prediction_horizon_days | INT           |
| forecast_amount         | DECIMAL(20,4) |
| lower_confidence_bound  | DECIMAL(20,4) |
| upper_confidence_bound  | DECIMAL(20,4) |
| confidence_score        | DECIMAL(5,4)  |
| trend_component         | DECIMAL(20,4) |
| seasonal_component      | DECIMAL(20,4) |
| residual_component      | DECIMAL(20,4) |
| forecast_type           | VARCHAR(50)   |
| generated_at            | TIMESTAMP     |

## Purpose

Predicts:

```text
Future spending
Future income
Future balances
Future category trends
```

---

# 3. pred_cash_flow_prediction

Predicts future cash flow and liquidity.

## Columns

| Column                | Type          |
| --------------------- | ------------- |
| prediction_key        | BIGINT (PK)   |
| model_key             | BIGINT (FK)   |
| user_key              | INT (FK)      |
| account_key           | INT (FK)      |
| prediction_date_key   | INT (FK)      |
| predicted_balance     | DECIMAL(20,4) |
| projected_income      | DECIMAL(20,4) |
| projected_expenses    | DECIMAL(20,4) |
| net_cash_flow         | DECIMAL(20,4) |
| liquidity_risk_score  | DECIMAL(5,4)  |
| overdraft_probability | DECIMAL(5,4)  |
| days_until_shortfall  | INT           |
| confidence_score      | DECIMAL(5,4)  |
| generated_at          | TIMESTAMP     |

## Purpose

Answers questions such as:

```text
Will the user run out of money?
How much cash will be available next week?
What is the risk of overdraft?
```

# ML Layer Relationship Overview

```text
pred_model_registry
        │
        ├─────────────┐
        │             │
        ▼             ▼
pred_time_series_forecast
pred_cash_flow_prediction
pred_behavioral_prediction
pred_anomaly_prediction
```

Supporting analytics:

```text
pred_tag_analysis
pred_feature_store
```

In total, the ML layer contains **7 tables**:

1. `pred_model_registry`
2. `pred_time_series_forecast`
3. `pred_cash_flow_prediction`
4. `pred_behavioral_prediction`
5. `pred_anomaly_prediction`
6. `pred_tag_analysis`
7. `pred_feature_store`

Together they support forecasting, anomaly detection, behavioral analysis, feature engineering, model management, and tag-based analytics on top of the core financial warehouse.
